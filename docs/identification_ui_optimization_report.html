参数辨识界面优化报告

本次优化围绕 S1 参数辨识流程、主界面实时精度刷新和健康状态可视化展开，保持本地单进程 PySide6 桌面工具链形态。

优化项

问题实现涉及文件
参数辨识过程耗时较长，缺少进度反馈 运行 S1 辨识时显示模态不确定进度弹窗，完成或失败后自动关闭。 app/pages/calibration_page.py
辨识界面返回主界面失败 返回逻辑改为沿父级链查找 MainWindow，适配页面被 QStackedWidget 包裹的结构。 app/pages/calibration_page.py
超差阈值修改后左侧显示不立即刷新 阈值输入右侧新增“更新”按钮，点击后立即重算当前构型定位误差、超差结论和健康状态。 app/pages/initialization_page.py
主页面右上角布局拥挤，UR 调试入口占用常用设置区 移除右侧设置区的 UR 调试按钮，改为顶部“调试”入口打开临时关节调试窗口。 app/pages/initialization_page.py
健康状态可视化不够直观 新增状态圆点与环形评分仪表盘：绿色表示 good，黄色表示 warning，红色表示 critical，灰色表示 unknown。 app/pages/initialization_page.py
