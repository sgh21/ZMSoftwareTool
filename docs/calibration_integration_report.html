3.2 修复"返回指南"按钮导致布局闪动

问题根因

原设计用两个独立按钮放在 GridLayout 不同列（col 1 和 col 2），通过 setVisible(True/False) 切换。当 "返回指南" 按钮 visible 状态变化时，GridLayout 重新计算 col 2 列宽，导致上方所有输入控件（模型路径编辑框、参数文件编辑框等）水平位移，产生明显"闪动"。

修复方案：单按钮动态切换

合并为一个按钮，始终占据同一位置（row 5, col 1），通过 setText() 改变文字、disconnect()/connect() 切换信号连接：

状态按钮文本连接方法外观
指南模式▶ 进入标定分析_toggle_calibration_panel蓝色 #calibration_button
标定模式← 返回指南_show_guide_panel蓝色 #calibration_button（一致）
def _toggle_calibration_panel(self):
    self._right_bottom_stack.setCurrentIndex(1)
    self.calibration_toggle_btn.setText("← 返回指南")
    self.calibration_toggle_btn.clicked.disconnect()
    self.calibration_toggle_btn.clicked.connect(self._show_guide_panel)

def _show_guide_panel(self):
    self._right_bottom_stack.setCurrentIndex(0)
    self.calibration_toggle_btn.setText("▶ 进入标定分析")
    self.calibration_toggle_btn.clicked.disconnect()
    self.calibration_toggle_btn.clicked.connect(self._toggle_calibration_panel)
效果：按钮始终在同一位置，文字变化不触发 GridLayout 重新布局，彻底消除闪动。两个方向的按钮样式一致（共用 #calibration_button 蓝色主题）。

四、标定置信度计算方式

定义位置

core/calibration_service.py — 函数 _confidence_from_cv() (第 800-811 行)

计算公式

confidence = 100.0 / (1.0 + cv_max_rmse_mm / (10.0 * limit_mm))

参数说明

参数来源含义
cv_max_rmse_mmS1 交叉验证最差折的拟合 RMSE (mm)交叉验证中表现最差那一折的均方根误差，衡量模型泛化能力
limit_mmconfig/thresholds.yaml → accuracy.position_rms_limit_mm（默认 0.5 mm）用户配置的定位精度容差上限

数值示例

CV 最差 RMSE (mm)limit_mm (mm)置信度
0.010.599.8%
0.10.598.0%
0.50.590.9%
1.00.583.3%
5.00.550.0%
注意：这不是贝叶斯后验概率。它是基于交叉验证误差的经验可靠性评分：CV 误差越小，置信度越接近 100%。初始化标定后置信度设定为 100%。

五、健康度（Health Score / Level）计算方式

定义位置

core/calibration_service.py — _score_from_error() 及 compute_predicted_position()

计算公式

ratio = error_mm / limit_mm
health_score = 100.0 / (1.0 + ratio / 3.0)
