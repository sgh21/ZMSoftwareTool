日期：2026-06-12。本文档为本次长期退化监控功能的手写总结，未通过脚本生成。追加更新：模型评估已增加进度条和后台线程，监控硬件参数已从模型结果 YAML 中拆分出来；置信度和总 RMS 已统一按模型不确定度计算。

模型定义

config/nominal_robot.yaml 是机器人定义模型，也是辨识模型的默认初始值。它保存辨识所需的名义 MD-H 参数、基坐标初值、靶球/工具偏置和关节限位；该文件不应被辨识后绝对模型覆盖。

identified_model 是辨识输出模型，保存在 storage/parameters/identified_model/。它包含最终绝对 MD-H、工具偏置、误差参数、拟合指标、初始定位不确定度和置信度历史。

controller_model 是控制器前向运动学基准，保存在 storage/parameters/controller_model/。实时“模型误差 RMS”定义为 p_identified(q) - p_controller(q) 的 RMS，而不是辨识模型与名义模型的差。

公式实现

服务层按 PDF 中的核心式实现：

Delta_T_i(k) = E_T_C * C_T_B(i,0) * inv(C_T_B(i,k)) * C_T_E

其中 E_T_C 为工具/末端到相机的手眼变换，C_T_E = inv(E_T_C)。随后使用 Log(Delta_T)^vee 得到 6 维残差，并输出位置漂移 RMS、姿态漂移 RMS 和综合等效漂移。

本次没有直接修改 MD-H 或误差参数。写回动作只更新模型置信度和 metrics.position_uncertainty_rmse_mm，并把旧 YAML 备份到 storage/model_versions/。

参数管理调整

旧的 calibration_result.yaml 只作为导入兼容格式；当前生效模型应从 config/active_parameters.yaml 找到对应的版本化参数文件。model_monitoring.yaml 的旧职责已迁移到 camera_monitoring 参数版本，服务层仍兼容旧版模型文件内嵌的 identification.monitoring 字段。

退化评估写回时只更新 active identified_model 中的 confidence、metrics.position_uncertainty_rmse_mm 和最近一次 last_degradation_evaluation 记录，不再把手眼或相机内参写回机器人模型文件。

置信度与总 RMS

辨识完成时，初始模型不确定度取最终辨识模型预测位置与激光跟踪仪真值之间的拟合残差 RMS：

u_initial_mm = fit_rmse_mm = rms(p_identified(q_i) - p_tracker(q_i))

初始置信度由该不确定度和超差阈值 L 计算：

confidence = 100 * L^2 / (L^2 + u_initial_mm^2)

模型校验时，固定标定板 PnP 观测得到当前位置退化 RMS：

u_degradation_mm = position_drift_rms_mm

写回模型时，不确定度按均方根合成：

position_uncertainty_rmse_mm = sqrt(u_previous_mm^2 + u_degradation_mm^2)

校验后的置信度不再用旧置信度乘衰减系数，而是用合成后的不确定度重新代入同一公式：

confidence_after = 100 * L^2 / (L^2 + position_uncertainty_rmse_mm^2)

实时界面的“模型误差 RMS”表示辨识模型和控制模型前向运动学位置差；“评估总 RMS”再合成当前模型不确定度：

model_rms_mm = rms(p_identified(q) - p_controller(q))

evaluated_rms_mm = sqrt(model_rms_mm^2 + position_uncertainty_rmse_mm^2)

当前边界

  • 本次实现的是加载式评估，不包含自动定时请求机器人运动和相机拍照。
  • 没有实现参数漂移反演；该部分需要补充模型参数雅可比和噪声协方差。
  • Mahalanobis 统计量在没有健康基线均值和协方差时会跳过，只输出 RMS 退化指标。
  • 图片目录 PnP 需要运行环境安装 opencv-python；项目推荐的 ZMSoftware 环境已验证可用。
  • 当前线程任务不提供取消按钮；若后续要支持取消，需要在 PnP 循环和服务层增加取消标志检查。
